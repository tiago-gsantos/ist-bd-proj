1)Endpoint /:
A função que usamos para executar no endpoint / é a lista clinicas que executa uma query select que lista
as informações pedidas de todas as clinicas

2)Endpoint: /c/<clinica>:
A função que usamos para executar neste endpoint é a listaEspecialidadesClinica que executa uma query select
que lista as especialidades desta clinica, consultando as tabelas medico,trabalha e clinica para este efeito

3) Endpoint: /c/<clinica>/<especialidade>:
A função que usamos para executar neste endpoint é a listaMedicosSlots que executa uma query select para
guardar num dicionario as informações dos médicos que trabalham nessa clínica e nessa especialidade 
(associando a cada chave nif,nome do médico a lista dos dias da semana em que este trabalha na
correspondente clinica e especialidade).
De seguida, para cada médico executamos outra query select que seleciona os primeiros 3 horários disponíveis
para marcação de consulta para esse medico na clinica e na especialidade correspondente (subtraindo a todos
os horários possíveis - tabela calendario filtrada pelos dias da semana correspondentes ao medico no dicionario -
aqueles em que ele já tem marcação)

4) Endpoint: /a/<clinica>/marcar
A função que usamos para executar neste endpoint é a marca_Consulta que executa após verificar os dados
de input (existencia do medico, do paciente e não existencia de consulta para o medico ou para o paciente
a essa hora e data ) uma query insert na tabela das consultas, adicionando uma consulta com esse paciente,
médico, clínica e horário.

5) Endpoint: /a/<clinica>/desmarcar
A função que usamos para executar neste endpoint é a cancelar_consulta que executa após verificar os dados
de input (existencia do medico, do paciente e existencia de consulta para o medico, paciente, horario e 
clinica passados no input) realiza uma transação onde se executam quatro queries. A primeira é um select 
que vai buscar o id e condigo_sns da consulta correspondente aos dados do input, a segunda para apagar todas
as observações associadas a essa consulta (tabela observação onde o id é o correspondente à consulta),
a terceira para apagar todas as receitas associadas à consulta (tabela receita onde o codigo_sns é o 
correspondente à consulta) e a quarta para apagar a propria consulta (tabela consulta com o id correspondente).
Note-se que não basta apagar a propria consulta, mas também todos os dados de outras tabelas associados a
essa consulta e que executamos estas quatro queries numa só transação para a nenhum momento termos dados
nas tabelas inconsistentes
